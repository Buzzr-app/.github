# Security Policy

## Reporting a Vulnerability

Please do not open public issues for security-sensitive reports.

Email security reports to Sarvesh or Rushil directly, or use GitHub private vulnerability reporting on public Buzzr repositories when available. Include:

- The affected repository and package or app area.
- Steps to reproduce the issue.
- Expected impact and any known exploit path.
- Relevant logs, screenshots, or proof-of-concept details.

We will acknowledge valid reports as quickly as possible, prioritize fixes by severity, and coordinate disclosure once the issue is resolved.

## Supported Projects

Security fixes are prioritized for actively maintained repositories in the `Buzzr-app` organization and published `@buzzr/*` npm packages.
