# Contributing to Buzzr

Thanks for helping improve Buzzr.

## Development Flow

1. Create a branch from `main`.
2. Keep changes focused and include tests for behavior changes.
3. Run the relevant local checks before opening a pull request.
4. Open a pull request and wait for CI plus review before merging.

## Expectations

- Do not commit secrets, tokens, credentials, or production data.
- Prefer small, reviewable pull requests.
- Keep user-facing behavior and package APIs documented when they change.
- Use clear commit messages and PR descriptions.

## Security

Report vulnerabilities privately using `SECURITY.md`. Do not disclose security issues in public issues or pull requests until a fix is released.
